# Техническое задание

## Трекер привычек для офисного сотрудника

---

## 1. Общая информация

**Название проекта:** Трекер привычек для офисного сотрудника

**Тип продукта:** Веб-интерфейс (дашборд)

**Целевая аудитория:** Люди, работающие в офисе, которые хотят вырабатывать полезные привычки без сложных настроек и перегруженного интерфейса.

**Ключевая идея:** Максимальная простота использования. Пользователь открывает дашборд, отмечает выполненные привычки и закрывает его — весь цикл занимает менее 5 секунд.

**Главный принцип:** Минимум действий — максимум пользы.

---

## 2. Цели и задачи

### Цель
Дать офисному сотруднику способ отслеживать простые полезные привычки в течение рабочего дня без когнитивной нагрузки и лишних настроек.

### Задачи
- Свести взаимодействие с интерфейсом к одному-двум касаниям на привычку
- Исключить необходимость регистрации, настройки профиля или сложных форм
- Сделать визуальный отклик мгновенным и понятным без пояснений
- Обеспечить, чтобы состояние (что уже отмечено сегодня) считывалось с первого взгляда

---

## 3. Функциональные требования

### 3.1 Основной сценарий использования
1. Пользователь открывает дашборд
2. Видит список привычек на сегодня с текущим статусом (выполнено/не выполнено)
3. Отмечает одним касанием выполненные пункты
4. Закрывает дашборд

Весь сценарий не должен требовать более одного касания на привычку и не должен включать промежуточные экраны, подтверждения или всплывающие окна.

### 3.2 Список привычек
- Готовый набор привычек по умолчанию, чтобы не заставлять пользователя настраивать список при первом запуске. Примеры:
  - Выпить воду
  - Прочитать 20 страниц
  - Прогулка
- Возможность добавить свою привычку (простая форма: название + сохранить)
- Возможность удалить привычку из списка
- Порядок привычек сохраняется между сессиями

### 3.3 Отметка выполнения
- Отметка происходит одним нажатием/тапом по привычке
- Визуальное состояние "выполнено" должно быть однозначным (не требует чтения текста, считывается по цвету/иконке)
- Повторное нажатие снимает отметку (отмена действия должна быть такой же простой, как и сама отметка)

### 3.4 Отображение прогресса
- Счётчик выполненных привычек за сегодня (например, "2 из 3")
- Индикатор серии дней подряд (стрик) — по каждой привычке или по общему выполнению дня
- Никаких обязательных к прочтению графиков на главном экране — детальная статистика (если есть) выносится на второй план, не мешает быстрому сценарию

### 3.5 Данные
- Состояние сохраняется автоматически, без кнопки "сохранить"
- Данные сохраняются локально/в хранилище, привязанном к пользователю
- При обновлении дня (наступлении нового календарного дня) отметки сбрасываются, история сохраняется

---

## 4. Требования к дизайну

### 4.1 Общий стиль
- Минимализм: минимум декоративных элементов, отсутствие лишних текстовых блоков и пояснений
- Современная графика: чистые формы, отсутствие теней, градиентов и визуального шума
- Фокус на одном экране — без вложенных разделов и многоуровневой навигации

### 4.2 Цветовая палитра
- Акцентные цвета: **зелёный** и **жёлтый**
- Зелёный — статус "выполнено"
- Жёлтый — акцент на активном/текущем действии, привлечение внимания к невыполненным пунктам
- Нейтральный фон и текст — без ярких дополнительных цветов, чтобы не перегружать восприятие

### 4.3 Типографика и компоновка
- Крупный, легко читаемый шрифт для названий привычек
- Достаточные отступы между элементами списка, чтобы исключить случайные нажатия
- Список привычек — главный и единственный обязательный элемент на экране

### 4.4 Адаптивность
- Интерфейс должен одинаково удобно работать на десктопе и мобильном устройстве, так как сценарий "открыть-отметить-закрыть" предполагает использование в течение дня, в том числе с телефона

---

## 5. Нефункциональные требования

- **Скорость загрузки:** дашборд должен быть готов к взаимодействию в течение 1-2 секунд после открытия
- **Отсутствие обязательной регистрации:** пользователь должен иметь возможность начать пользоваться инструментом сразу, без создания аккаунта (регистрация может быть опциональной для сохранения данных между устройствами)
- **Доступность:** достаточный цветовой контраст между статусами "выполнено"/"не выполнено" с учётом того, что зелёный и жёлтый выбраны акцентными

---

## 6. Вне рамок проекта (не входит в ТЗ)

- Социальные функции (совместные списки, сравнение с коллегами)
- Push-уведомления и напоминания
- Интеграции с внешними календарями или таск-трекерами
- Расширенная аналитика и отчёты за длительные периоды

---

## 7. Критерии приёмки

- Полный цикл "открыть дашборд → отметить все выполненные привычки → закрыть" занимает менее 5 секунд для пользователя, уже знакомого с интерфейсом
- Новый пользователь понимает, как отметить привычку, без пояснений и подсказок
- Статус "выполнено" считывается с первого взгляда, без необходимости кликать или наводить курсор
- Интерфейс не содержит ни одного обязательного действия сверх отметки привычек
